⚖️ 資料同步平衡原則

GDriveSync ── 本地 / Google Drive / NAS 三方同步規則
🕐
原則一：最新版本優先（mtime 勝出）
當多方同時修改同一檔案時，以 修改時間 (mtime) 最新的版本為權威來源， 並自動覆蓋所有落後端點。時間精度依賴 Google Drive UTC 時間戳與本地/NAS 系統時鐘同步。
🗑️
原則二：刪除優先（三方全刪）
任一方（本地、Drive 或 NAS）執行刪除操作，即觸發三方同步刪除。 即使另一方同時修改了該檔案，刪除仍優先，以確保空間一致性與版本整潔性。
🌐
原則三：NAS 離線降級（雙方繼續）
當 NAS（192.168.120.31）無法連線時，系統不中斷， 自動降級為本地 ↔ Google Drive 雙向同步模式，並於日誌寫入 [WARNING] 標記。 NAS 恢復後，下次同步將自動補齊缺漏資料。
🔐
原則四：MD5 完整性驗證（避免假同步）
每次同步前以 MD5 雜湊值 比對各端檔案內容， 若 MD5 相同則跳過傳輸，避免因時間戳差異導致的無效覆寫，節省頻寬與 API 配額。
📋
原則五：日誌透明（24 小時滾動保留）
所有同步動作（新增、更新、刪除、衝突解決、錯誤）均寫入 sync.log， 並透過 GitHub Actions 自動推送至雲端。日誌保留最近 24 小時的紀錄， 超過者自動清除，確保儲存空間不膨脹。本儀表板即時呈現此日誌狀態。
